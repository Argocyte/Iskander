## 1. MISSION STATEMENT
Implement the "Stewardship Council" (SC) logic. The SC is a dynamic, federated layer of governance for the Iskander network. It is NOT a permanent board of directors; it is an emergent, recallable delegation layer activated when inter-node contracts fail or competitive external threats arise.

## 2. CORE ARCHITECTURAL CONCEPTS
- **Recursive Governance:** Governance logic must support Layer 1 (Node) -> Layer 2 (Council) -> Layer 3 (Constitutional Core).
- **gSBT (Governance Soulbound Tokens):** Non-transferable tokens representing voting weight.
- **Impact Score (The Scoring Engine):** A dynamic reputation metric calculated by the OS:
  `Impact_Score = (Historical_Contribution_Value / Ecosystem_Total_Value) * Ethical_Audit_Score`
- **Liquid Delegation:** Users delegate their gSBT weight to "Stewards." This delegation is instantaneous, revocable, and only permitted if the Steward has an `Impact_Score` above the protocol-defined threshold.

## 3. TECHNICAL REQUIREMENTS
### A. The Delegation Ledger (Smart Contract - Solidity)
- **Constraint:** Must support gas-efficient batch updates for when a node's `Impact_Score` shifts due to REA-accounting updates.
- **Functions required:**
  - `delegate(address steward)`: Validates Steward's `Impact_Score` via Chainlink/Iskander-Oracle before accepting delegation.
  - `revoke()`: Instantly sets delegation weight back to self.
  - `getVotingWeight(address node)`: Returns current active weight based on self-contribution + received delegations.
- **Integration:** The contract must be able to read updates from the `TreasuryAgent` (via an Oracle or Merkle Root update provided by the OS).

### B. The Scoring Engine (Python / FastAPI / LangGraph)
- **Task:** Create the `StewardshipScorer` class.
- **Logic:**
  - Aggregates contribution logs from the `Valueflows` database.
  - Fetches `Ethical Audit` results (from the Ethical Auditor Agent).
  - Normalizes the `Impact Score` (0.0 to 1.0).
  - Proposes/Updates the "Steward Threshold" periodically.

### C. HITL & Constitutional Safeguards
- **Veto-Right:** Implement an `EmergencyVeto` function. Any Node can initiate a veto if they provide a "Rationale Proof" (Glass Box Protocol) citing a violation of the 10 CCIN Principles.
- **Circuit Breaker:** If `Total_Escrow_Value` (on-chain) > `Fiat_Reserve_Value` * `Solvency_Factor`, the system must prevent Council decisions that would increase financial exposure.

## 4. IMMEDIATE TASKS FOR AGENT
1. **Define the Interface:** Write the `IStewardshipLedger` solidity interface.
2. **Implement the Scoring Logic:** Draft the Python `StewardshipScorer` logic.
3. **Draft the "Glass Box" Protocol:** Define how the Council must submit its "Rationale" for any cross-node decision (required for auditability).
4. **Safety Check:** Ensure that `revoke()` is always executable regardless of network congestion or contract state.

## 5. DESIGN PRINCIPLES
- **No Hierarchical Permanence:** Roles expire automatically if the user's `Impact_Score` drops below the threshold.
- **FOSS-First:** All code must be Apache 2.0 or AGPL-3.0 compatible.
- **Anticipatory:** The system should warn nodes when their delegation status is about to expire due to dropping contribution scores.
